# TLV 标准实现说明

## 概述

本实现遵循 **BER (Basic Encoding Rules)** 标准的 TLV 编码格式，支持可变长度的 Type 和 Length 字段。

## TLV 结构

```
┌──────────┬──────────┬──────────┐
│   Type   │  Length  │  Value   │
├──────────┼──────────┼──────────┤
│  1-2字节  │  1-4字节  │  可变长度 │
└──────────┴──────────┴──────────┘
```

## 编码规则

### 1. Type 字段（1-2 字节）

- **单字节 Type**：`0x00` - `0x7F`
  - 直接使用一个字节表示
  - 例如：`0x01`, `0x02`, `0x7F`

- **双字节 Type**：`0x80` - `0xFFFF`
  - 使用两个字节表示（Big Endian）
  - 例如：`0x8001`, `0xFFFF`

### 2. Length 字段（BER 编码）

#### 短格式（1 字节）- 长度 < 128

```
┌─┬─────────┐
│0│ 7 bits  │  直接表示长度值（0-127）
└─┴─────────┘
```

- 示例：`0x0D` = 13 字节

#### 长格式（2-5 字节）- 长度 >= 128

```
第一字节：
┌─┬─────────┐
│1│ 后续字节数 │  高位为 1，低 7 位表示后续有几个字节
└─┴─────────┘

后续字节：
└─ 实际长度值（Big Endian）
```

- **1 字节长度** (128-255)：`0x81 LL`
  - 例如：`0x81 C8` = 200 字节

- **2 字节长度** (256-65535)：`0x82 HH LL`
  - 例如：`0x82 04 00` = 1024 字节

- **3 字节长度** (65536-16777215)：`0x83 HH MM LL`
  - 例如：`0x83 01 00 00` = 65536 字节

- **4 字节长度** (>16777215)：`0x84 HH HH LL LL`
  - 例如：`0x84 00 01 86 A0` = 100000 字节

### 3. Value 字段

- 根据 Length 字段指定的长度读取
- 可以是任意二进制数据

## 编码示例

### 示例 1：短数据（单字节 Type + 短格式 Length）

```
数据: "Hello"
Type: 0x01
Length: 5

编码结果:
┌────┬────┬────────────────────────────┐
│ 01 │ 05 │ 48 65 6C 6C 6F             │
├────┼────┼────────────────────────────┤
│Type│Len │  H  e  l  l  o              │
└────┴────┴────────────────────────────┘
```

### 示例 2：中等数据（单字节 Type + 长格式 Length）

```
数据: 200 字节的数据
Type: 0x02
Length: 200 (0xC8)

编码结果:
┌────┬────┬────┬────────────┐
│ 02 │ 81 │ C8 │ ... data...│
├────┼────┼────┼────────────┤
│Type│长格式│200 │  200字节   │
└────┴────┴────┴────────────┘
```

### 示例 3：大数据（单字节 Type + 2字节长度）

```
数据: 1024 字节的数据

编码结果:
┌────┬────┬────┬────┬────────────┐
│ 03 │ 82 │ 04 │ 00 │ ... data...│
├────┼────┼────┼────┼────────────┤
│Type│2字节│  1024   │ 1024字节   │
└────┴────┴────┴────┴────────────┘
```

### 示例 4：双字节 Type

```
数据: "Complex Type"

编码结果:

```

## API 使用

### 创建和序列化

```swift
// 创建 TLV
let value = "Hello, World!".data(using: .utf8)!
let tlv = AETlv(type: 0x01, value: value)

// 序列化
let serialized = tlv.serialize()
```

### 反序列化

```swift
// 单个 TLV
if let (tlv, consumed) = AETlv.deserialize(data) {
    print("Type: 0x\(String(format: "%02X", tlv.type))")
    print("Length: \(tlv.length)")
    print("Value: \(tlv.value)")
}

// 多个 TLV
let tlvs = AETlv.deserializeMultiple(data)
for tlv in tlvs {
    // 处理每个 TLV
}
```

## 加密协商中的应用

### Tag 定义

```swift
0x01 - 加密类型 (ECC/RSA)
0x02 - 协议版本 (1.0)
0x03 - 公钥数据
0x04 - 会话 ID
0x05 - 时间戳
...
```

### 协商请求示例

```

│加密类型│3字节│ 数据  │版本│3字节│数据 │公钥│32字节│  数据 │

```

## 优势

1. **可扩展性**：支持任意长度的数据
2. **标准化**：遵循 BER 标准，易于互操作
3. **灵活性**：Type 可以是 1 或 2 字节
4. **效率**：短数据使用短格式，减少开销
5. **兼容性**：与 ASN.1、X.509 等标准兼容
